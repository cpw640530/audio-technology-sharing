# 音频技术分享知识大纲

本文档与当前网页内容保持同步，用于说明“音频技术分享”网站的知识分类、主题卡片和实验室建设方向。网页当前以“知识分类大纲 + 可搜索主题卡片 + 主题详情 + 交互实验室”为主要结构，覆盖音频基础、硬件、软件、传统算法、AI 音频和应用场景。

## 1. 当前网页结构

- 首页标题：音频技术分享 / Audio Technology Sharing。
- 首页说明：一个可搜索、可筛选、带交互实验室的音频技术分享网页。
- 左侧区域：知识分类大纲，按分类列出所有主题。
- 右侧主内容：搜索框、分类筛选、主题卡片。
- 主题详情：详细解释、实验室入口、关键知识点、关键概念、常见误区、内容扩展建议。
- 实验室页面：每个重点卡片对应独立可视化或文章式实验室。

## 2. 音频基础

### 2.1 什么是声音

- 解释声音的物理本质：振动、介质传播、声波。
- 逐条解释频率、振幅、相位、波长、周期等概念。
- 通过交互式正弦波说明三角函数参数如何影响听到的声音。
- 对应实验室：声音波形实验室，可调频率、振幅、相位并观察波形变化。

### 2.2 数字音频基础

- 解释模拟信号如何变成数字音频。
- 重点说明采样、量化、编码三个步骤。
- 展示 PCM 如何组织采样值，以及 PCM 加上信息头如何形成 WAV。
- 对比 MP3、AAC、AMR、Opus、ADPCM 等编码格式的压缩思路和适用场景。
- 对应实验室：采样、量化与编码实验室。

### 2.3 听感与指标

- 解释响度、频响、失真、噪声、动态、声像等听感概念。
- 覆盖 LUFS、SNR、THD、动态范围等客观指标。
- 说明指标和主观听感之间的关系，以及为什么单一指标不能代表全部听感。
- 对应实验室：听感与指标实验室，展示噪声、失真、声像、动态处理等波形变化。

## 3. 音频硬件

### 3.1 麦克风

- 介绍模拟驻极体咪头、数字 MEMS 麦、动圈话筒和多麦阵列。
- 说明灵敏度、信噪比、指向性、频响和拾音距离。
- 对比模拟麦与数字麦的输出方式和系统接入差异。
- 对应实验室：麦克风指向性与拾音实验室。

### 3.2 ADC / DAC / Codec

- 说明 ADC 采集、DAC 重建和 Codec 芯片链路的区别。
- 区分模拟输入、数字 PCM、I2S/TDM/PDM、功放和扬声器之间的关系。
- 解释位深、采样率、量化阶梯、时钟和链路流程图。
- 对应实验室：ADC / DAC / Codec 实验室。

### 3.3 数字音频接口和传输协议

- 介绍 I2S、TDM、PDM、SPDIF、USB Audio 的基本用途。
- 说明常见线数、时钟、帧同步、通道和数据线关系。
- 重点补充：
  - I2S 的 BCLK、LRCLK、SD、MCLK。
  - TDM 的 slot 和 frame sync。
  - PDM 如何由高速 1-bit 流抽取滤波得到 PCM。
  - SPDIF 和 USB Audio 的典型使用场景。
- 对应实验室：数字音频接口实验室。

### 3.4 功放与扬声器

- 解释 Class A、Class AB、Class D 功放的基本原理和效率差异。
- 说明扬声器单元、箱体、分频器、功率、阻抗和灵敏度。
- 展示功放、扬声器和匹配关系如何影响声音输出。
- 对应实验室：功放与扬声器实验室。

## 4. 音频软件

### 4.1 系统音频架构

- 以 Linux / ALSA 等系统为具体例子说明音频软件层次。
- 区分应用、中间件、驱动、硬件、Codec、I2S、功放和扬声器。
- 同时展示采集数据流和播放数据流。
- 对应实验室：系统音频架构实验室。

### 4.2 实时音频处理

- 解释采集、处理、编码、传输、解码、播放的实时链路。
- 说明 ALSA 环形 buffer、copy to user、filter、codec、RTSP/网络传输、jitter buffer 等环节。
- 补充回采播放数据用于 AEC 回声消除的流程。
- 对应实验室：实时音频处理实验室。

### 4.3 音频编程与插件

- 介绍音频插件、宿主、实时回调、参数自动化和音频图。
- 说明插件开发中实时线程、内存分配、延迟补偿和旁路处理的注意点。
- 对应实验室：音频编程与插件实验室。

### 4.4 音频编解码

- 解释 PCM、WAV、MP3、AAC、AMR、Opus、ADPCM 等格式。
- 分开展示编码流程和解码流程。
- 说明音乐存储、在线会议、语音通信等不同场景如何选择编码格式。
- 对应实验室：音频编解码实验室。

## 5. 传统算法

### 5.1 基础信号处理

- 解释 FFT 是快速傅里叶变换，STFT 是短时傅里叶变换。
- 说明窗口长度、hop size、频率 bin、频谱图和能量图的含义。
- 展示滤波器 / EQ 对频响和输出波形的影响。
- 解释动态处理中的阈值、ratio、输入电平和增益计算。
- 对应实验室：基础信号处理实验室。

### 5.2 语音增强

- 解释 AEC、AGC、ANR/NS、Dereverb 的基本原理。
- 区分单麦增强和多麦增强：多麦下才有 DOA 声源定位和定向拾音。
- 补充 SoC SDK 常见音频处理流程，例如瑞芯微、君正、SigmaStar 等链路组织方式。
- 说明滑动窗口、噪声、回声、混响拖尾强度如何影响波形展示。
- 对应实验室：语音增强实验室。

### 5.3 空间音频

- 解释 ITD、ILD、HRTF、房间反射、头部朝向和对象音频。
- 对比耳机双耳渲染、5.1/7.1、Atmos、游戏和车载座舱空间音频。
- 说明头部朝向变化时，声源相对角度、左右耳差异和反射表现需要同步变化。
- 对应实验室：空间音频实验室。

## 6. AI 音频

### 6.1 AI 音频总流程

- 从麦克风 / Codec / ADC 采集得到 PCM 数字音频。
- 将 PCM 经过分帧、FFT/STFT、Mel 频谱、MFCC 或 embedding 转成模型输入特征。
- 模型根据任务执行识别、生成、增强、分类或编码。
- 说明不同任务不会都走完全相同的模型路径。
- 对应实验室：AI 音频总流程实验室。

### 6.2 语音识别 ASR

- 解释 PCM 输入、分帧加窗、log-mel/MFCC、Encoder、CTC/RNNT、解码与文本后处理。
- 补充离线转写、流式字幕、会议纪要、车载语音等场景。
- 常见模型：Conformer / RNNT、wav2vec 2.0、Whisper、Paraformer / FunASR、GPT-4o transcribe。
- 对应实验室：语音识别 ASR 实验室。

### 6.3 语音合成 TTS

- 解释文本规范化、音素/韵律预测、声学模型、声码器和波形输出。
- 应用包括语音播报、有声书、智能客服、车载提示、数字人和多语言配音。
- 对应实验室：语音合成 TTS 实验室。

### 6.4 音频生成

- 介绍音乐生成、音效生成、声音补全和多模态音频生成。
- 说明 prompt、条件控制、生成质量、版权和安全边界。
- 对应实验室：音频生成实验室。

### 6.5 AI 音频事件识别

- 识别哭声、玻璃破碎、狗叫声、警报声等非语音事件。
- 说明音频事件识别通常使用 log-mel 频谱图和分类模型。
- 常见模型：YAMNet、PANNs、AST、BEATs、CLAP。
- 对应实验室：AI 音频事件识别实验室。

### 6.6 AI 音频增强

- 用神经网络估计干净语音、频谱掩蔽或目标声源。
- 应用于会议、耳机、直播降噪、车载通话、助听和短视频修音。
- 说明 AI 增强和传统 AEC/NS 的边界，以及音乐噪声、断字、过度抑制等风险。
- 对应实验室：AI 音频增强实验室。

### 6.7 AI 音频编码

- 介绍 SoundStream、EnCodec、Lyra、codec token / neural codec language model。
- 区分神经音频编码和传统 MP3/AAC 的手工变换编码路线。
- 对应实验室：AI 音频编码实验室。

## 7. 应用场景

### 7.1 会议与通信

- 解释上行采集、下行播放、回采参考、AEC、NS/ANR、AGC、编码、网络、Jitter Buffer 和字幕链路。
- 典型问题诊断：回声大、听不清、字幕慢。
- 对应实验室：会议与通信实验室。

### 7.2 车载声学

- 聚焦车内麦克风、扬声器、车载语音助手、声源定位、座舱空间音频和主动降噪 ANC/RNC。
- 通过车内饰部件位置图展示顶灯麦克风阵列、门板扬声器、中置扬声器、ANC 误差麦、语音助手和空间音频。
- 典型问题诊断：唤醒错人、定位漂移、ANC 压耳或轰鸣。
- 对应实验室：车载声学实验室。

### 7.3 IoT 与内容创作

- 将 IoT 语音控制和内容创作生产路径放在同一个应用主题中对比。
- IoT 侧重点：低功耗监听、本地唤醒词、命令识别、意图权限、设备动作、语音反馈。
- 内容创作侧重点：录音 / 导入、降噪 / 清理、剪辑 / 修音、压缩 / 混音、母带 / 响度、多平台导出。
- 典型问题诊断：误唤醒、直播声音忽大忽小、AI 配音不自然。
- 对应实验室：IoT 与内容创作实验室。

## 8. 后续扩展原则

- 新增或扩展主题卡片时，默认需要加入对应实验室。
- 实验室应接入 `TopicLab` 类型、卡片 metadata、独立 React 组件、`App.tsx` 视图、`TopicDetails.tsx` 打开逻辑和测试。
- 页面说明优先使用图解、流程图、波形、频谱或交互控件，不只做静态文字堆叠。
- 对容易混淆的概念要补充关键知识点、常见误区和应用边界。
- 每次修改后运行测试、构建和基本页面访问检查。
