# 功放与扬声器实验室设计

## 目标

扩展“功放与扬声器”主题，形成一套通用基础科普内容，并新增独立“功放与扬声器实验室”。首版重点不是工程计算器，而是通过图解和可听音效说明：小信号如何经过功放变成功率输出，扬声器如何把电能转成空气声波，以及功放、扬声器、箱体、分频和保护之间如何共同影响听感。

## 范围

本阶段采用“内容增强 + 独立实验室首版”方案。

包含：

- 扩展“功放与扬声器”卡片详情。
- 新增“功放与扬声器实验室”入口。
- 实验室支持图解切换和 5 个可听音效模式。
- 使用浏览器 Web Audio 生成演示音效，不依赖外部音频文件。
- 支持中文和英文界面文本。

不包含：

- 精确扬声器箱体仿真。
- 真实 SPL、温升、功放 SOA 或电池续航计算。
- 麦克风录音、外部音频文件上传或真实设备测量。
- 多声道家庭影院、汽车功放拓扑等大型系统专题。

## 卡片内容结构

“功放与扬声器”卡片详情扩展为 4 个层次。

### 功放是什么

功放负责把 DAC、Codec 或前级输出的小信号变成能推动负载的电压、电流和功率。内容需要解释：

- 输入信号和增益。
- 电源电压与最大输出摆幅。
- 输出功率、电流能力和负载。
- 效率、散热和电池供电限制。
- 削波、过载、过流、过温和保护。

### 功放类型

覆盖 Class A、Class AB、Class D：

- Class A：线性好，效率低，静态功耗和发热大。
- Class AB：传统音频功放常见，在效率和失真之间折中。
- Class D：用开关/PWM 方式工作，效率高，适合手机、蓝牙音箱、电视、智能音箱、车载和便携设备。需要说明开关噪声、输出滤波、布局和 EMI。

### 扬声器如何发声

以动圈扬声器为主线，解释音圈、磁路、振膜、悬边和定心支片。电流经过音圈产生电磁力，推动振膜往复运动，带动空气形成声波。

需要覆盖的关键概念：

- 阻抗。
- 灵敏度。
- 额定功率与峰值功率。
- 频率响应。
- 低频下潜。
- 非线性失真。
- 振膜行程和热限制。

### 系统匹配

强调功放和扬声器不能只看单个参数：

- 功放功率要和扬声器承受能力匹配。
- 阻抗越低，同电压下电流越大，对功放和散热要求越高。
- 小扬声器低频能力有限，需要箱体、DSP、动态保护和限幅配合。
- 分频器把低频、中频、高频送给适合的单元。
- 瓦数更大不等于一定更响或更好听，灵敏度、失真、箱体和保护策略同样重要。

## 实验室页面

新增独立页面“功放与扬声器实验室”。入口放在“功放与扬声器”卡片详情中，视觉位置和已有麦克风、ADC/DAC/Codec、数字音频接口实验室保持一致。

实验室采用三块布局：

- 信号链图：展示 `DAC/Codec 输出 -> 功放 -> 分频/保护 -> 扬声器单元 -> 空气声波`。
- 可视化图解：根据当前模式显示不同图。
- 音效演示与参数：选择 5 个音效模式，调整强度并播放/停止。

### 图解模式

首版支持 4 个图解模式：

- 功放类型：切换 Class A、Class AB、Class D，重点显示线性放大、交越区、PWM/开关输出和滤波。
- 扬声器单元：展示音圈、磁路、振膜运动和空气声波。
- 箱体与分频：展示低音/高音单元、分频点、箱体低频响应和共振。
- 匹配关系：展示功率、阻抗、灵敏度和保护之间的关系。

### 音效模式

首版支持 5 个音效模式，每个模式有弹窗说明“现象、原因、听感、常见场景、改善方式”。

- 削波失真：对波形做硬削波，强度越大越刺耳。
- 谐波失真：加入 2 次和 3 次谐波，模拟非线性。
- 低频不足：用高通或低频衰减模拟小扬声器没低频。
- 箱体共振：用窄带峰值滤波突出某个频率，产生“嗡”“闷”或单频突出的听感。
- 动态保护/限幅：用压缩和限幅模拟小音箱为了防止破音、过热或过行程而压低峰值。

## 音频播放约束

实验室使用 Web Audio API 生成演示音效。为了避免多个音源叠加，必须满足：

- 页面只维护一个当前播放句柄。
- 切换音效、切换模式、点击停止或离开页面时，先停止并断开旧节点。
- 播放按钮状态和当前音效状态保持同步。
- 不自动播放，必须由用户点击触发。
- 如果浏览器不支持 Web Audio，显示降级提示，但图解和知识内容仍可使用。

## 组件边界

建议新增 `AmplifierSpeakerLab` 组件，避免继续扩大现有实验室组件。

主要职责：

- 管理当前图解模式、功放类型、音效模式和强度。
- 渲染信号链、图解 SVG、参数控件和音效说明弹窗。
- 管理 Web Audio 播放生命周期。

知识数据仍放在 `src/content/knowledge.ts`，通过现有主题详情组件展示。

路由方式沿用当前 `App.tsx` 的 `activeView` 模式，新增实验室视图即可。

## 测试要求

新增或扩展测试覆盖：

- “功放与扬声器”卡片详情包含新增的功放、扬声器和系统匹配知识点。
- 卡片详情中存在“打开功放与扬声器实验室”入口。
- 实验室标题、返回按钮和核心信号链文本正常显示。
- 能切换 Class A、Class AB、Class D、扬声器单元、箱体与分频、匹配关系。
- 5 个音效模式都能打开说明弹窗。
- 切换音效时旧播放状态被清理，不保留多个播放状态。
- 中文关键文本正常显示。

## 验收标准

- 用户能从“功放与扬声器”卡片进入独立实验室。
- 普通读者能通过图解理解“信号 -> 功率 -> 振膜 -> 声波”的链路。
- 用户能听到 5 类典型问题的差异。
- 页面风格和已有音频实验室一致。
- `npm test -- --run src/App.test.tsx` 通过。
- `npm run build` 通过。
